## Overview

This tutorial walks through the full **zap-in** flow -- adding liquidity to a Meteora pool (DLMM or DAMM V2) using the LP Agent Open API:

1. **Discover** a pool using the pool discovery endpoint
2. **Fetch pool info** to get the active bin and price range
3. **Generate** unsigned zap-in transactions
4. **Sign** the transactions locally with your wallet
5. **Land** the signed transactions on-chain via Jito bundles

## Prerequisites

- An LP Agent API key (get one from the [API Dashboard](https://app.lpagent.io/open-api))
- A Solana wallet keypair
- Node.js >= 18 and the following packages:

```bash
npm install @solana/web3.js bs58
```

## Setup

```typescript
import { Keypair, Transaction, VersionedTransaction } from "@solana/web3.js";
import bs58 from "bs58";

const API_BASE = "https://api.lpagent.io/open-api/v1";
const API_KEY = "your-api-key-here";

// Load your wallet keypair
const PRIVATE_KEY = "your-base58-private-key";
const wallet = Keypair.fromSecretKey(bs58.decode(PRIVATE_KEY));
const OWNER = wallet.publicKey.toBase58();

// Helper for API calls
async function apiCall(method: string, path: string, body?: object) {
  const res = await fetch(`${API_BASE}${path}`, {
    method,
    headers: {
      "Content-Type": "application/json",
      "x-api-key": API_KEY,
    },
    body: body ? JSON.stringify(body) : undefined,
  });

  if (!res.ok) {
    const error = await res.json();
    throw new Error(`API error: ${JSON.stringify(error)}`);
  }

  return res.json();
}
```

## Step 1: Discover a Pool

Use the pool discovery endpoint to find a suitable pool. You can filter by market cap, TVL, volume, and more.

```typescript
const discoverRes = await apiCall("GET",
  "/pools/discover?" + new URLSearchParams({
    chain: "SOL",
    sortBy: "vol_24h",
    sortOrder: "desc",
    pageSize: "5",
    min_market_cap: "5",       // min $5M market cap
    min_liquidity: "50",       // min $50K TVL
  }).toString()
);

const pools = discoverRes.data;

// Pick the first pool
const pool = pools[0];
console.log(`Pool: ${pool.pool}`);
console.log(`Pair: ${pool.token0_symbol}/${pool.token1_symbol}`);
console.log(`TVL: $${pool.tvl.toLocaleString()}`);
console.log(`24h Volume: $${pool.vol_24h.toLocaleString()}`);
console.log(`Protocol: ${pool.protocol}`); // "meteora" or "meteora_damm_v2"
```

## Step 2: Get Pool Info

Fetch the pool details to find the active bin ID and decide your liquidity range.

```typescript
const poolId = pool.pool;
const poolInfoRes = await apiCall("GET", `/pools/${poolId}/info`);

const poolInfo = poolInfoRes.data;
const activeBin = poolInfo.liquidityViz?.activeBin;

console.log(`Active Bin ID: ${activeBin?.binId}`);
console.log(`Current Price: ${activeBin?.pricePerToken}`);
console.log(`Pool Type: ${poolInfo.type}`); // "meteora" or "meteora_damm_v2"
```

Use the active bin to define your liquidity range. A common approach is to place liquidity around the current price:

```typescript
const RANGE = 34; // Number of bins on each side of active bin
const fromBinId = activeBin.binId - RANGE;
const toBinId = activeBin.binId + RANGE;

console.log(`Liquidity range: bin ${fromBinId} to ${toBinId} (${RANGE * 2 + 1} bins)`);
```

## Step 3: Generate Zap-In Transactions

Call the `add-tx` endpoint to generate unsigned transactions. The API auto-detects whether the pool is DLMM or DAMM V2.

### Zap-in mode (recommended)

The simplest way -- provide SOL and the API handles swapping to the correct token ratio automatically:

```typescript
const addTxRes = await apiCall("POST", `/pools/${poolId}/add-tx`, {
  stratergy: "Spot",           // "Spot" | "Curve" | "BidAsk"
  inputSOL: 0.1,               // Amount of SOL to deposit
  percentX: 0.5,               // 50% allocation to token X
  fromBinId,
  toBinId,
  owner: OWNER,
  slippage_bps: 500,           // 5% slippage
  provider: "JUPITER_ULTRA",   // Swap provider
  mode: "zap-in",
});
```

### Manual mode (provide both tokens)

If you already hold both tokens and want to skip the swap step:

```typescript
const addTxRes = await apiCall("POST", `/pools/${poolId}/add-tx`, {
  stratergy: "Spot",
  amountX: 0.05,               // Amount of token X
  amountY: 7.5,                // Amount of token Y
  fromBinId,
  toBinId,
  owner: OWNER,
  slippage_bps: 500,
  mode: "normal",
});
```

**Response:**

## Step 4: Sign the Transactions

The API returns base64-encoded transactions that need to be signed with your wallet before landing.

```typescript
function signTransaction(base64Tx: string): string {
  const buffer = Buffer.from(base64Tx, "base64");

  // Try to deserialize as VersionedTransaction first, then legacy Transaction
  try {
    const tx = VersionedTransaction.deserialize(buffer);
    tx.sign([wallet]);
    return Buffer.from(tx.serialize()).toString("base64");
  } catch {
    const tx = Transaction.from(buffer);
    tx.partialSign(wallet);
    return tx
      .serialize({ requireAllSignatures: false, verifySignatures: false })
      .toString("base64");
  }
}

const { lastValidBlockHeight, swapTxsWithJito, addLiquidityTxsWithJito, meta } = addTxRes.data;

// Sign all swap transactions
const signedSwapTxs = swapTxsWithJito.map(signTransaction);

// Sign all add liquidity transactions
const signedAddTxs = addLiquidityTxsWithJito.map(signTransaction);

console.log(`Signed ${signedSwapTxs.length} swap txs and ${signedAddTxs.length} add liquidity txs`);
```

## Step 5: Land the Transactions

Submit the signed transactions via the `landing-add-tx` endpoint. This sends them as Jito bundles for faster inclusion.

```typescript
const landRes = await apiCall("POST", "/pools/landing-add-tx", {
  lastValidBlockHeight,
  swapTxsWithJito: signedSwapTxs,
  addLiquidityTxsWithJito: signedAddTxs,
  meta,
});

console.log(`Method: ${landRes.data.method}`);       // "JITO"
console.log(`Signature: ${landRes.data.signature}`);  // Transaction signature
console.log(`View on Solscan: https://solscan.io/tx/${landRes.data.signature}`);
```

## Complete Example

Here's the full flow in a single script:

```typescript
import { Keypair, Transaction, VersionedTransaction } from "@solana/web3.js";
import bs58 from "bs58";

const API_BASE = "https://api.lpagent.io/open-api/v1";
const API_KEY = "your-api-key-here";
const wallet = Keypair.fromSecretKey(bs58.decode("your-base58-private-key"));
const OWNER = wallet.publicKey.toBase58();

async function apiCall(method: string, path: string, body?: object) {
  const res = await fetch(`${API_BASE}${path}`, {
    method,
    headers: { "Content-Type": "application/json", "x-api-key": API_KEY },
    body: body ? JSON.stringify(body) : undefined,
  });
  if (!res.ok) throw new Error(`API error: ${await res.text()}`);
  return res.json();
}

function signTx(base64Tx: string): string {
  const buffer = Buffer.from(base64Tx, "base64");
  try {
    const tx = VersionedTransaction.deserialize(buffer);
    tx.sign([wallet]);
    return Buffer.from(tx.serialize()).toString("base64");
  } catch {
    const tx = Transaction.from(buffer);
    tx.partialSign(wallet);
    return tx.serialize({ requireAllSignatures: false, verifySignatures: false }).toString("base64");
  }
}

async function main() {
  // 1. Discover a pool
  const discover = await apiCall("GET", "/pools/discover?" + new URLSearchParams({
    chain: "SOL", sortBy: "vol_24h", sortOrder: "desc", pageSize: "5",
  }));
  const pool = discover.data[0];
  console.log(`Found pool: ${pool.token0_symbol}/${pool.token1_symbol} (${pool.pool})`);

  // 2. Get pool info for active bin
  const info = await apiCall("GET", `/pools/${pool.pool}/info`);
  const activeBin = info.data.liquidityViz?.activeBin;
  const fromBinId = activeBin.binId - 34;
  const toBinId = activeBin.binId + 34;
  console.log(`Active bin: ${activeBin.binId}, range: ${fromBinId}-${toBinId}`);

  // 3. Generate add liquidity transactions
  const addTx = await apiCall("POST", `/pools/${pool.pool}/add-tx`, {
    stratergy: "Spot",
    inputSOL: 0.1,
    percentX: 0.5,
    fromBinId,
    toBinId,
    owner: OWNER,
    slippage_bps: 500,
    mode: "zap-in",
  });
  console.log(`Generated ${addTx.data.addLiquidityTxsWithJito.length} add txs`);

  // 4. Sign all transactions
  const signedSwapTxs = addTx.data.swapTxsWithJito.map(signTx);
  const signedAddTxs = addTx.data.addLiquidityTxsWithJito.map(signTx);

  // 5. Land the transactions
  const result = await apiCall("POST", "/pools/landing-add-tx", {
    lastValidBlockHeight: addTx.data.lastValidBlockHeight,
    swapTxsWithJito: signedSwapTxs,
    addLiquidityTxsWithJito: signedAddTxs,
    meta: addTx.data.meta,
  });

  console.log(`Success! Tx: https://solscan.io/tx/${result.data.signature}`);
}

main().catch(console.error);
```

## Strategy Types

| Strategy | Description |
|----------|-------------|
| `Spot` | Uniform distribution across all bins in the range. Best for stable pairs. |
| `Curve` | Bell curve distribution concentrated around the active bin. Best for range-bound markets. |
| `BidAsk` | Heavier allocation at the edges of the range. Best for volatile pairs where you expect mean reversion. |

## Tips

- **Bin range**: A wider range (more bins) means less impermanent loss risk but lower fee APR. Start with 30-70 bins on each side.
- **Slippage**: Use 300-500 bps (3-5%) for most pairs. Increase for low-liquidity tokens.
- **Zap-in mode**: Recommended for simplicity. Just provide SOL and the API handles swapping to the correct token ratio automatically.
- **Transaction expiry**: Transactions expire after `lastValidBlockHeight`. Generate and land them promptly -- don't wait more than ~60 seconds.
- **DAMM V2 pools**: The `add-tx` endpoint auto-detects DAMM V2 pools. The same zap-in flow works for both pool types -- no code changes needed.

## Error Handling

Common errors and how to handle them:

| Error | Cause | Fix |
|-------|-------|-----|
| `Token X price unavailable` | Price feed missing for one of the tokens | Try again later or use a different pool |
| `Insufficient balance` | Not enough tokens in your wallet | Reduce `inputSOL` or `amountX`/`amountY` |
| `Missing amount to ZapIn` | `inputSOL` not provided in zap-in mode | Add `inputSOL` to the request body |
| Transaction expired | `lastValidBlockHeight` has passed | Re-generate and re-sign the transactions |
